# Versioning

ForgeWire ships **four independently-versioned artifacts**. They are *not*
kept in lockstep — each moves when its own surface changes — but each must be
internally consistent (source == built == installed == what the running
process reports). This document defines what each line governs, how to bump
it, and how to verify the running cluster reflects the bump.

## The four version lines

| Artifact | Version source(s) | Reported at runtime by | Current |
|---|---|---|---|
| **Rust daemons** (hub, runner, loom-runner, cli, protocol) | `Cargo.toml` `[workspace.package] version` (all crates `version.workspace = true`) | Hub `GET /healthz` → `package_version` / `version` | `0.9.0` |
| **Python package** `forgewire-fabric` (hub server, MCP sidecars, runner sidecar) | `pyproject.toml` `version` **and** `python/forgewire_fabric/__init__.py` `__version__` — keep these two equal | `importlib.metadata.version("forgewire-fabric")`; all four MCP servers' `serverInfo.version` (via `Server(name, version=__version__)`) | `0.18.0` |
| **VS Code extension** (VSIX) | `vscode/package.json` `version` | VS Code extension host / marketplace | `0.6.1` |
| **Wire protocol** | `PROTOCOL_VERSION` constants (Rust `fabric-protocol`, Python MCP servers) | Hub `/healthz` → `protocol_version`; MCP handshake | `4` |

> The `forgewire-runtime` PyO3 wheel (`crates/fabric-py`) carries its own
> dynamic version and is gated by `RUNTIME_COMPAT` in `__init__.py`; bump that
> envelope when the Rust workspace major/minor moves.

## Why independent, not unified

The Rust runtime and the Python surface change at different rates. Forcing one
number would either inflate the Rust line for Python-only churn or pin Python
behind a stable runtime. Each line is **monotonic** — never move a version
*down* to "align" with another artifact.

## Bump rules (pre-1.0 semver)

All artifacts are `0.x`, so:

- **patch** (`0.x.Y+1`) — fixes, internal refactors, doc/build-only changes.
- **minor** (`0.X+1.0`) — new features **or breaking changes** (no major while
  in `0.x`). Removing an endpoint, changing a wire contract, or hard-rejecting
  a previously-accepted input is a minor bump.
- **major** (`1.0.0`) — only when the surface is declared stable.

Bump **protocol_version** separately, and only when the on-wire contract
changes in a way peers must agree on. A Python or Rust release does **not**
imply a protocol bump.

### Recent history

- Rust `0.8.0 → 0.9.0` (Jun-12): DHCP-proof hostname addressing.
- Python `0.16.0 → 0.17.0` (Jun-10, M2.9): loom signing, signed stdin, hub parity.
- Python `0.17.0 → 0.18.0` (Jun-13, M2.8.9/M2.8.10 + identity): removed MCP
  shims and `/tasks/claim-v2`, hard-reject missing `kind` (breaking dispatch
  surface), cross-runtime runner-identity schema support.

## Pin → rebuild → reinstall → verify

"Pinned, no regression" means: after bumping a source version, the **built and
installed** artifacts must be exactly that version, and the running process
must report it. Source-only bumps that aren't reinstalled cause the silent
drift this scheme exists to prevent (e.g. dist metadata lagging `__version__`).

### Rust
```
cargo build --release -p fabric-hub -p fabric-runner -p fabric-cli -p loom-runner
# deploy binaries, reinstall services (scripts/install/install-fabric.ps1)
# verify:
curl -s http://127.0.0.1:8765/healthz   # package_version == Cargo workspace version
```

### Python
Bump **both** `pyproject.toml` and `__init__.py` to the same value, then
reinstall so `importlib.metadata` and MCP `serverInfo` stop reporting the old
dist:
```
python -m pip install -e . --no-deps
python -c "import importlib.metadata as m; print(m.version('forgewire-fabric'))"  # == __version__
# MCP servers report it automatically via Server(name, version=__version__)
```
A stale `forgewire_fabric.egg-info` in the tree (gitignored) is regenerated by
the reinstall; delete it if metadata ever resolves to an old value.

### VSIX
```
cd vscode && npm run compile && npx vsce package   # filename carries the version
```

## Verification checklist (run after any release rebuild)

- [ ] `GET /healthz` `package_version` == `Cargo.toml` workspace version (both nodes)
- [ ] `importlib.metadata.version("forgewire-fabric")` == `__init__.__version__` == `pyproject` version
- [ ] All four MCP servers' `serverInfo.version` == Python `__version__`
- [ ] `/healthz` `protocol_version` matches the `PROTOCOL_VERSION` constants
- [ ] VSIX filename/`package.json` version matches the installed extension
